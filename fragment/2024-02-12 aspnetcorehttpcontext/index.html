如何在 ASP.NET Core 中访问 HttpContext

net

ASP.NET Core 中已经把 HttpContext.Current 删掉，取而代之的做法就是将 IHttpContextAccessor 通过依赖注入的方式注入到你的服务中，然后从这个类中导出 HttpContext，参考代码如下：

public class MyComponent : IMyComponent
{
    private readonly IHttpContextAccessor _contextAccessor;

    public MyComponent(IHttpContextAccessor contextAccessor)
    {
        _contextAccessor = contextAccessor;
    }

    public string GetDataFromSession()
    {
        return _contextAccessor.HttpContext.Session.GetString(*KEY*);
    }
}

解决方法：

将 IHttpContextAccessor 注入到静态类中，让后续操作回到你的 ASP.NET MVC 时代，参考如下代码：

public static class HttpHelper
{
     private static IHttpContextAccessor _accessor;
     public static void Configure(IHttpContextAccessor httpContextAccessor)
     {
          _accessor = httpContextAccessor;
     }

     public static HttpContext HttpContext => _accessor.HttpContext;
}

然后将 IHttpContextAccessor 赋值到 Startup.Configure 方法中。

HttpHelper.Configure(app.ApplicationServices.GetRequiredService<IHttpContextAccessor>());

最后将 IHttpContextAccessor 以单例模式灌到 Services IOC 中。

services.AddSingleton<IHttpContextAccessor, HttpContextAccessor>();

其实在 ASP.NET Core 中一切都是依赖注入先行，这是所有事情的起点，如果你用的是静态，那还注入个啥

在 ASP.NET MVC 中的 HttpContext.Current 有诸多的缺点，显而易见的就是不利单元测试。

HttpContext.Current 在asp.net 中多线程中，存在不同线程中，此值为空的情况，只有在启动线程时，把值注入，但要注意在线程退出时，进行清理，避免上下文环境污染。
