端口占用排查

linux windows

在网站开发、部署的过程经常出现端口不能使用的问题，要想知道端口是由什么程序占用，通过几个命令联合使用，很容易解决。

Windows下端口占用查询

windows中出现端口不能使用，有两种情形：

  • 端口已使用
  • 端口已保留

端口已使用

在windows下查询端口是否被使用，可以使用如下命令：

C:\Windows\System32>netstat -ano | findstr "5015"
  TCP    0.0.0.0:5015           0.0.0.0:0              LISTENING       16252
  TCP    127.0.0.1:4413         127.0.0.1:5015         ESTABLISHED     3912
  TCP    127.0.0.1:5015         127.0.0.1:4413         ESTABLISHED     16252

从上面的命令中，可以看出端口 5015 被pid = 16252,3912 的应用占用

查看占用进行名称：

C:\Windows\System32>tasklist | findstr "3912"
msedge.exe                    3912 Console                    1     50,764 K

C:\Windows\System32>tasklist | findstr "16252"
node.exe                     16252 Console                    1    209,012 K

初步定位为：控制台启动的node.exe占用了此端口

如果要关闭此应用可以通过如下方法：

taskkill /F /PID 16252

端口已保留

在 Windows 中，某些端口范围可能被系统保留用于特定服务，例如动态端口分配给 Hyper-V 虚拟机、Windows 更新服务、或其他网络服务。这个分配的行为是动态、随机的，所以会出现用的好好的，但突然不可用的情况。

查询端口保留的情况：

C:\Windows\System32>netsh interface ipv4 show excludedportrange protocol=tcp

协议 tcp 端口排除范围

上面列出的端口范围也是不能使用的。
